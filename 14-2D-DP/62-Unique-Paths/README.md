# Unique Paths
**LeetCode**: #62 | **Difficulty**: Medium |
**Category**: 2-D Dynamic Programming

## 🔍 Problem Statement
機器人位於一個 `m x n` 網格的左上角。機器人每次只能向下或向右移動一步。

機器人試圖到達網格的右下角。問總共有多少條不同的路徑？

**Example 1:**
```
Input: m = 3, n = 7
Output: 28
```

**Example 2:**
```
Input: m = 3, n = 2
Output: 3
Explanation:
從左上角到右下角共有 3 條路徑：
1. 右 -> 下 -> 下
2. 下 -> 下 -> 右
3. 下 -> 右 -> 下
```

## 💡 解題思路

### 關鍵洞察
- **核心觀察**：到達某格的路徑數 = 從上方到達的路徑數 + 從左方到達的路徑數
- **問題特徵**：二維 DP，可以分解為重疊的子問題

### 演算法選擇
- **為什麼選 2-D DP？**
  - 遞迴法：會重複計算，時間 O(2^(m+n))
  - **2-D DP**：儲存中間結果，時間 O(m * n)

- **適用場景**
  - 當需要在二維網格中計數路徑時
  - 當移動受限（只能向右、向下）時
  - 複雜度分析：時間 O(m * n)，空間 O(m * n)（可優化到 O(n))

### 時機判斷
看到以下特徵時，考慮使用 2-D DP：
- 🗺️ 關鍵字：「路徑數」、「網格」、「只能向右/下」
- 📊 結構：二維網格，移動方向受限
- ♻️ 模式：問題可以分解為相同的子問題

## 🎯 實作細節

### 步驟分解
1. **初始化 DP 表**：`dp[m][n]`
2. **基准情況**：
   - 第一行和第一列都只有 1 條路徑（只能一直向右或向下）
3. **填充 DP 表**：
   - 對於每個格 `(i, j)`：
     - `dp[i][j] = dp[i-1][j] + dp[i][j-1]`
4. **返回結果**：`dp[m-1][n-1]`

### 關鍵程式碼片段
```python
def uniquePaths(m, n):
    # 初始化 DP 表
    dp = [[1] * n for _ in range(m)]

    # 填充 DP 表
    for i in range(1, m):
        for j in range(1, n):
            dp[i][j] = dp[i-1][j] + dp[i][j-1]

    return dp[m-1][n-1]
```

**為什麼這樣寫？**
- 第一行和第一列都初始化為 1（只有一條路徑）
- 每個格子的路徑數等於上方和左方路徑數之和
- 最終結果在右下角 `dp[m-1][n-1]`

### 易錯點
⚠️ **坑點1**：忘記初始化第一行和第一列
⚠️ **坑點2**：索引錯誤（off by one）- 注意是從 1 開始
⚠️ **坑點3**：使用大數組可以優化到一維 DP

## 📊 複雜度分析

| 方法 | 時間複雜度 | 空間複雜度 | 優點 | 缺點 |
|------|------------|------------|------|------|
| 組合數學 | O(min(m, n)) | O(1) | 極快 | 需要數學知識 |
| **2-D DP** | O(m * n) | O(m * n) | **直觀** | 可以優化空間 |
