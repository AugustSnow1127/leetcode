# Climbing Stairs
**LeetCode**: #70 | **Difficulty**: Easy |
**Category**: 1-D Dynamic Programming

## 🔍 Problem Statement
假設你正在爬樓梯。需要 n 階才能到達頂部。

每次你可以爬 1 或 2 個階梯。有多少種不同的方法可以爬到頂部？

**Example 1:**
```
Input: n = 2
Output: 2
Explanation: 有兩種方法：
1. 1 階 + 1 階
2. 2 階
```

**Example 2:**
```
Input: n = 3
Output: 3
Explanation: 有三種方法：
1. 1 階 + 1 階 + 1 階
2. 1 階 + 2 階
3. 2 階 + 1 階
```

## 💡 解題思路

### 關鍵洞察
- **核心觀察**：到達第 n 階的方法數 = 到第 n-1 階的方法數 + 到第 n-2 階的方法數
- **問題特徵**：明顯的重疊子問題和最優子結構

### 演算法選擇
- **為什麼選 Dynamic Programming？**
  - 遞迴法：會重複計算，時間 O(2^n)
  - **DP**：儲存中間結果，時間 O(n)

- **適用場景**
  - 當問題可以分解為重疊的子問題時
  - 當需要避免重複計算時
  - 複雜度分析：時間 O(n)，空間 O(1)（空間優化版本）

### 時機判斷
看到以下特徵時，考慮使用 DP：
- 📊 關鍵字：「多少種方法」、「計數」、「最優解」
- ♻️ 模式：可以分解為相同的子問題
- 🔄 重疊：會重複計算相同的子問題

## 🎯 實作細節

### 步驟分解
1. **基准情況**：
   - n = 1：1 種方法（1 階）
   - n = 2：2 種方法（1+1 或 2）
2. **DP 轉移方程**：
   - `dp[i] = dp[i-1] + dp[i-2]`
   - 類似 Fibonacci 序列
3. **空間優化**：
   - 只需要前兩個值，不需要整個數組
   - 用兩個變數：`prev2, prev1`

### 關鍵程式碼片段
```python
def climbStairs(n):
    if n <= 2:
        return n

    prev2, prev1 = 1, 2  # dp[1], dp[2]
    for i in range(3, n + 1):
        current = prev1 + prev2
        prev2 = prev1
        prev1 = current

    return prev1
```

**為什麼這樣寫？**
- 空間優化：不需要整個 dp 數組，只用 O(1) 空間
- 從 3 開始迭代，已經處理了基准情況
- 每次迭代更新兩個變數，保持滾動窗口

### 易錯點
⚠️ **坑點1**：忘記處理 n <= 2 的基准情況
⚠️ **坑點2**：使用 O(n) 空間 - 可以優化到 O(1)
⚠️ **坑點3**：遞迴沒有記憶化 - 會超時

## 📊 複雜度分析

| 方法 | 時間複雜度 | 空間複雜度 | 優點 | 缺點 |
|------|------------|------------|------|------|

## 🔑 關鍵模式
**1-D DP 基本模板**
```python
dp = [0] * (n + 1)
dp[0] = 1  # 基准情況
for i in range(1, n + 1):
    dp[i] = 某個計算
return dp[n]
```
